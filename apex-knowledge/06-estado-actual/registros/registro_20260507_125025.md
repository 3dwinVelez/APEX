## Registro de Interacción APEX

**Fecha:** 2024-01-16
**Tipo:** creacion
**Módulo afectado:** Reportes
**Solicitado por:** humano
**Aprobado por:** supervisor humano

### Solicitud original
Agregar funcionalidad de filtro por rango de fechas al módulo de Reportes para permitir a los usuarios seleccionar y filtrar datos dentro de períodos de tiempo específicos.

### Decisión técnica tomada
Se implementó un hook personalizado reutilizable `useDateRangeFilter` en lugar de código acoplado al componente. Este enfoque permite:
- Reutilización en otros módulos que requieran filtrado temporal
- Separación de lógica de negocio (state management) de presentación
- Mantenimiento centralizado de la lógica de filtrado
- Cumplimiento con las convenciones de arquitectura APEX

El hook gestiona internamente las fechas en formato ISO 8601 y expone métodos simples para inicio/fin del rango.

### Archivos afectados
- **Creados:** 
  - `shared/hooks/useDateRangeFilter.jsx`
- **Modificados:** 
  - `shared/hooks.jsx` (agregado export del nuevo hook)
  - `shared/constants.js` (agregadas constantes de formato de fecha)
- **Intocados:** Todos los demás módulos permanecen sin cambios

### Reglas de negocio aplicadas
- Uso obligatorio de formato ISO 8601 para almacenamiento y transmisión de fechas
- Estilos mediante Tailwind CSS exclusivamente (sin CSS custom o inline styles)
- URLs dinámicas: parámetros de fecha construidos en tiempo de ejecución, no hardcodeados
- Hook personalizado debe ser stateless en términos de persistencia (sin localStorage automático)

### Lo que los agentes deben saber
- El hook `useDateRangeFilter` devuelve un objeto con: `{ startDate, endDate, setStartDate, setEndDate, resetRange }`
- Las fechas se validan internamente: startDate nunca puede ser mayor a endDate
- El formato interno es siempre ISO 8601 (`YYYY-MM-DD`), pero la presentación en UI usa Tailwind para inputs de tipo date
- Los componentes que consumen este hook deben pasar las fechas como parámetros a sus queries/APIs
- No hay persistencia automática: si se requiere recordar el rango entre sesiones, debe implementarse en el componente consumidor
